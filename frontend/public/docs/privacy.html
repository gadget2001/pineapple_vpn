Политика обработки персональных данных Pineapple VPN

Версия: 2026-03-15 · Дата вступления в силу: 15.03.2026

1. Общие положения

Настоящая Политика определяет порядок обработки персональных данных пользователей сервиса Pineapple VPN. Документ применяется к Telegram-боту, Telegram MiniApp и backend API сервиса.

Используя ключевые функции сервиса (trial, подписка, конфигурация, платежи), пользователь подтверждает ознакомление с настоящей Политикой.

2. Оператор и контакты

Оператор организует обработку данных для предоставления защищенного удаленного доступа к российским сервисам из-за границы. По вопросам персональных данных пользователь обращается через поддержку в Telegram-боте / MiniApp Pineapple VPN.

3. Какие данные обрабатываются

3.1. Данные Telegram-профиля

  • Telegram ID;
  • username;
  • first_name/last_name (если переданы Telegram);
  • технические атрибуты авторизации WebApp.

3.2. Данные использования сервиса

  • дата/время входа и действий в интерфейсе;
  • IP-адрес подключения;
  • audit-логи и технические журналы;
  • этап onboarding, выбор ОС, факт завершения;
  • факт, дата и версия принятия юридических документов.

3.3. Платежные и подписочные данные

  • история пополнений, списаний и начислений;
  • статус и сроки подписки, выбранный тариф;
  • технические идентификаторы платежей и webhook-метаданные.

Оператор не хранит полные реквизиты банковских карт. Платежные реквизиты обрабатываются платежным провайдером.

3.4. VPN-данные

  • служебный UUID профиля;
  • subscription URL;
  • служебные технические параметры, необходимые для выдачи и поддержки конфигурации.

3.5. Реферальные данные

  • реферальный код;
  • связи приглашений;
  • история и объем начислений по реферальной программе.

4. Цели обработки

  1. авторизация пользователя через Telegram;
  2. оказание услуг Pineapple VPN;
  3. проведение onboarding и фиксация согласий;
  4. активация trial и учет подписки;
  5. обработка платежей и операций внутреннего кошелька;
  6. выдача и поддержка VPN-конфигурации;
  7. работа реферальной программы;
  8. обеспечение безопасности и предотвращение злоупотреблений;
  9. поддержка пользователей и диагностика;
  10. выполнение требований законодательства.

5. Правовые основания обработки

  • исполнение договора (оферты) с пользователем;
  • законный интерес оператора по обеспечению стабильной и безопасной работы сервиса;
  • исполнение требований законодательства;
  • согласие пользователя — в предусмотренных случаях.

6. Хранение данных и инфраструктура

Основное хранение данных осуществляется во внешней PostgreSQL. Вспомогательные состояния и очереди обрабатываются через Redis и служебные компоненты backend/worker/scheduler.

7. Передача данных третьим лицам

Передача данных допускается только в объеме, необходимом для работы сервиса:

  • Telegram — как платформа взаимодействия;
  • платежный провайдер (например, ЮKassa) — для платежных операций;
  • инфраструктурные подрядчики — для хостинга и сопровождения;
  • уполномоченные органы — в случаях, установленных законом.

Персональные данные не продаются и не передаются в маркетинговые базы третьих лиц.

8. Сроки хранения

  • логи подключений: в пределах срока конфигурации сервиса (по умолчанию 30 дней);
  • платежные и подписочные данные: в сроки, необходимые для учета, аудита и правовой защиты;
  • данные о принятии документов: для подтверждения акцепта и согласия.

По достижении целей обработки данные удаляются или обезличиваются, если закон не требует иного.

9. Права пользователя

  1. получать сведения об обработке данных;
  2. требовать исправления неточных данных;
  3. требовать удаления данных при отсутствии законных оснований для хранения;
  4. отзывать согласие в применимых случаях;
  5. обращаться в уполномоченный орган и суд для защиты прав.

10. Меры защиты

  • разграничение прав доступа к данным;
  • журналирование критичных действий;
  • ограничение прямого доступа к инфраструктуре;
  • мониторинг технических событий и инцидентов.

11. Изменение Политики

Оператор вправе обновлять Политику. Новая редакция публикуется в сервисе с указанием версии и даты вступления в силу.
