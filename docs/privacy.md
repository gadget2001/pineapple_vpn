# Политика обработки персональных данных Pineapple VPN

Версия: 2026-03-15  
Дата вступления в силу: 15.03.2026

## 1. Общие положения
Настоящая Политика обработки персональных данных (далее — Политика) определяет порядок и условия обработки персональных данных пользователей сервиса Pineapple VPN.

Политика распространяется на все основные каналы сервиса:
- Telegram-бот Pineapple VPN;
- Telegram MiniApp Pineapple VPN;
- backend API и связанные служебные компоненты (worker/scheduler, сервисные интеграции).

Использование ключевого функционала сервиса (в том числе активация пробного периода, пополнение кошелька, оформление подписки, получение VPN-конфигурации) означает, что пользователь ознакомился с настоящей Политикой и согласен с ее условиями в применимой части.

## 2. Оператор и контакт по вопросам данных
Оператор персональных данных в рамках Pineapple VPN организует и осуществляет обработку персональных данных пользователей для оказания услуг защищенного удаленного доступа к российским сервисам из-за границы.

Канал для обращений по вопросам персональных данных, удаления данных и реализации прав субъекта данных: поддержка сервиса в Telegram-боте и Telegram MiniApp.

## 3. Категории обрабатываемых данных
В зависимости от действий пользователя сервис может обрабатывать следующие категории данных.

### 3.1. Данные Telegram-профиля
- Telegram ID;
- username;
- first_name, last_name (если переданы Telegram);
- технические атрибуты Telegram WebApp initData, необходимые для авторизации.

### 3.2. Данные аккаунта и использования сервиса
- дата и время регистрации/входа;
- статусы onboarding-процесса (этап, выбранная ОС, завершение);
- факт, дата и версия акцепта юридических документов;
- IP-адрес подключения к сервису;
- сервисные журналы действий (audit events), необходимые для безопасности, диагностики и антиабуза.

### 3.3. Платежные и подписочные данные
- сведения о пополнениях, списаниях, начислениях и статусах операций;
- история подписки (тариф, сроки, статус);
- технические идентификаторы платежей и webhook-метаданные платежного провайдера.

Важно: сервис не хранит полные реквизиты банковских карт пользователей. Платежные реквизиты обрабатываются платежным провайдером в его защищенном контуре.

### 3.4. Данные VPN-профиля
- служебные идентификаторы VPN-профиля (включая UUID);
- subscription URL пользователя;
- технические параметры, требуемые для выдачи и обслуживания конфигурации.

### 3.5. Данные реферальной программы
- реферальный код пользователя;
- факт перехода приглашенного пользователя по реферальной ссылке;
- связи «пригласивший / приглашенный»;
- начисления и история реферальных выплат.

### 3.6. Данные обращений в поддержку
- содержание сообщений пользователя в поддержку;
- служебные данные по инцидентам и времени обработки обращений.

## 4. Цели обработки персональных данных
Обработка персональных данных осуществляется исключительно для легитимных целей работы Pineapple VPN, включая:

1. авторизацию пользователей через Telegram и управление сессией;
2. предоставление доступа к функциям сервиса;
3. выполнение onboarding-сценария и фиксацию согласий;
4. активацию пробного периода и учет условий trial;
5. обработку пополнений, списаний, оформления и продления подписок;
6. выдачу, обновление и обслуживание VPN-конфигурации;
7. работу реферальной системы;
8. выполнение антифрод-проверок, антиабуз-контроля и мер безопасности;
9. диагностику, поддержку, обработку инцидентов и улучшение качества сервиса;
10. выполнение обязательных требований законодательства.

## 5. Правовые основания обработки
Правовыми основаниями обработки персональных данных являются:
- исполнение договора (оферты) между оператором и пользователем;
- законный интерес оператора по обеспечению безопасности и стабильности сервиса;
- необходимость исполнения требований законодательства;
- согласие пользователя в тех случаях, где оно требуется.

## 6. Принципы обработки
Оператор придерживается следующих принципов:
- законность, добросовестность и прозрачность обработки;
- ограничение обработки достижением конкретных целей;
- минимизация объема данных и недопустимость избыточной обработки;
- актуальность и точность данных в разумных пределах;
- ограничение сроков хранения;
- обеспечение конфиденциальности и безопасности.

## 7. Условия хранения и инфраструктура
Базовые данные пользователей хранятся во внешней PostgreSQL базе данных.

Вспомогательные состояния и очереди задач могут обрабатываться через Redis и сервисные компоненты backend/worker/scheduler.

Обработка данных может осуществляться в инфраструктуре хостинг-провайдера, используемого оператором для размещения компонентов Pineapple VPN.

## 8. Передача данных третьим лицам
Передача персональных данных третьим лицам допускается только в объеме, необходимом для работы сервиса и в рамках заявленных целей.

К таким получателям могут относиться:
- Telegram как платформа взаимодействия с ботом и MiniApp;
- платежный провайдер (например, ЮKassa) для приема и подтверждения платежей;
- инфраструктурные подрядчики (хостинг/серверная инфраструктура/техническое обслуживание);
- уполномоченные государственные органы — в случаях и порядке, предусмотренных законом.

Оператор не продает персональные данные пользователей и не передает их в маркетинговые базы третьих лиц.

## 9. Сроки хранения данных
Срок хранения данных определяется целями обработки, требованиями законодательства и настройками сервиса.

Ориентировочно:
- логи подключений и технические журналы — в пределах срока, заданного конфигурацией (по умолчанию 30 дней, если иное не установлено);
- данные подписки, платежей и финансовых операций — в период, необходимый для расчетов, аудита, урегулирования споров и соблюдения требований закона;
- сведения о согласии с документами — на период использования сервиса и/или срок, необходимый для подтверждения факта акцепта;
- данные аккаунта — до удаления аккаунта и/или до истечения обязательных сроков хранения.

По истечении сроков хранения данные удаляются или обезличиваются, если дальнейшее хранение не требуется законом.

## 10. Права пользователя
Пользователь как субъект персональных данных имеет право:
1. получать информацию об обработке своих данных;
2. требовать уточнения, блокирования или удаления неточных/неактуальных данных;
3. требовать удаления данных при отсутствии законных оснований для их дальнейшей обработки;
4. отозвать согласие в случаях, когда обработка основана именно на согласии;
5. обжаловать действия оператора в уполномоченный орган или в судебном порядке.

Для реализации прав необходимо обратиться в поддержку Pineapple VPN.

## 11. Меры защиты данных
Оператор применяет разумные и достаточные организационные и технические меры защиты, включая:
- разграничение прав доступа к данным;
- журналирование административных и критичных действий;
- защиту API и сервисных контуров авторизацией;
- ограничение прямого доступа к внутренней инфраструктуре;
- регулярный мониторинг инцидентов и технических ошибок.

При этом ни одна система не может гарантировать абсолютную защиту от всех возможных угроз, но оператор принимает меры для снижения рисков.

## 12. Трансграничные аспекты
Сервис предназначен для пользователей, которые могут находиться за пределами РФ. В связи с этим часть сетевых взаимодействий может иметь трансграничный характер в пределах технической архитектуры Telegram и интернет-инфраструктуры.

Оператор стремится минимизировать передачу данных и ограничивать ее необходимым объемом.

## 13. Данные несовершеннолетних
Сервис Pineapple VPN не ориентирован на несовершеннолетних. Пользователь подтверждает, что обладает необходимой дееспособностью для акцепта документов и использования платных функций.

## 14. Изменение Политики
Оператор вправе изменять Политику в одностороннем порядке.

Актуальная версия публикуется в интерфейсе сервиса с указанием даты и версии документа.

При существенных изменениях, влияющих на права пользователей, в сервисе может запрашиваться повторное подтверждение ознакомления с документами.

## 15. Применимое право
К отношениям, связанным с обработкой персональных данных в Pineapple VPN, применяется право Российской Федерации в части, подлежащей применению.
