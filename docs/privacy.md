# Политика обработки персональных данных Pineapple VPN

Версия: 2026-03-15  
Дата вступления в силу: 15.03.2026

## 1. Общие положения
Настоящая Политика описывает, какие данные обрабатываются в сервисе Pineapple VPN, для каких целей и на каких основаниях. Политика применяется к Telegram-боту, Telegram MiniApp и связанным API сервиса.

Используя Pineapple VPN, пользователь подтверждает, что ознакомился с этой Политикой до начала использования ключевого функционала (активация trial, получение конфигурации, оплата).

## 2. Оператор и каналы связи
Оператор сервиса Pineapple VPN обрабатывает данные для предоставления защищенного удаленного доступа к российским сервисам из-за границы.

По вопросам обработки данных пользователь может обратиться через поддержку в Telegram-боте/ MiniApp (кнопка «Поддержка»).

## 3. Какие данные мы обрабатываем
В зависимости от сценария использования сервис обрабатывает:

1. Данные Telegram-аккаунта:
- Telegram ID
- username
- first_name / last_name (если переданы Telegram)

2. Данные использования сервиса:
- IP-адрес подключения
- дата и время подключения
- журнал действий в сервисе (audit events)
- состояние onboarding (этап, выбор ОС, факт завершения)
- факт и дата принятия юридических документов, версия документов

3. Данные подписки и платежей:
- сведения о тарифе, статусе подписки, сроках действия
- история пополнений и списаний
- идентификаторы и метаданные платежей (в том числе от платежного провайдера)

4. Технические данные VPN-профиля:
- служебный UUID профиля
- subscription URL
- технические параметры, необходимые для выдачи конфигурации

5. Данные реферальной программы:
- реферальный код
- сведения о приглашениях
- начисления реферального вознаграждения

Мы не запрашиваем специальные категории персональных данных и биометрические данные.

## 4. Цели обработки данных
Данные обрабатываются для:

1. регистрации и авторизации пользователя через Telegram;
2. предоставления доступа к функциям Pineapple VPN;
3. активации trial-периода и учета подписки;
4. обработки платежей, зачислений и списаний;
5. выдачи и поддержки VPN-конфигурации;
6. работы реферальной программы;
7. обеспечения безопасности сервиса, предотвращения злоупотреблений;
8. поддержки пользователей и разбора инцидентов;
9. выполнения требований применимого законодательства.

## 5. Правовые основания обработки
Основания обработки:

1. исполнение договора с пользователем (предоставление сервиса);
2. законный интерес оператора в обеспечении безопасности и стабильной работы;
3. выполнение обязанностей, установленных законодательством;
4. согласие пользователя в части, где оно требуется.

## 6. Где и как хранятся данные
- Основная база данных: внешняя PostgreSQL.
- Вспомогательные данные/состояния: Redis.
- Компоненты сервиса: backend, bot, frontend, worker/scheduler и связанные инфраструктурные сервисы.

Доступ к данным предоставляется только в объеме, необходимом для работы соответствующих компонентов.

## 7. Передача данных третьим лицам
Данные могут передаваться:

1. платежному провайдеру для проведения оплаты (например, ЮKassa);
2. Telegram как платформе взаимодействия с ботом и MiniApp;
3. инфраструктурным подрядчикам/хостинг-провайдерам в пределах технической необходимости;
4. уполномоченным органам в случаях и порядке, предусмотренных законом.

Мы не продаем персональные данные.

## 8. Сроки хранения
- Логи подключений хранятся в пределах срока, установленного конфигурацией сервиса (по умолчанию 30 дней).
- История платежей, подписок и финансовых операций хранится в пределах срока, необходимого для бухгалтерских, правовых и операционных целей.
- Данные акцепта юридических документов хранятся для подтверждения факта согласия.

После достижения целей обработки данные удаляются или обезличиваются, если иное не требуется законом.

## 9. Права пользователя
Пользователь вправе:

1. запросить сведения об обработке своих данных;
2. запросить исправление неточных данных;
3. запросить удаление данных, если нет законных оснований для дальнейшего хранения;
4. отозвать согласие в части, где обработка основана на согласии;
5. обратиться с жалобой в уполномоченный орган при наличии оснований.

Для реализации прав используйте поддержку сервиса.

## 10. Безопасность
Оператор применяет разумные организационные и технические меры защиты данных, включая:

- разграничение доступа;
- контроль и аудит действий;
- ограничение прямого доступа к инфраструктуре;
- мониторинг и журналирование событий безопасности.

## 11. Изменение Политики
Мы можем обновлять Политику. Новая редакция публикуется в сервисе с указанием версии и даты.

При существенных изменениях пользователь знакомится с актуальными документами повторно до продолжения использования ключевых функций.

## 12. Контакты
По вопросам обработки персональных данных и реализации прав пользователя обращайтесь через поддержку в Telegram-боте/ MiniApp.
