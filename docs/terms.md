Версия: 2026-03-15  
Дата вступления в силу: 15.03.2026

## 1. Статус документа
Настоящий документ является публичной офертой на заключение договора возмездного оказания услуг Pineapple VPN.

Пользователь акцептует оферту путем совершения действий в интерфейсе сервиса: подтверждение согласия с документами и/или начало использования оплачиваемых функций (пополнение кошелька, покупка тарифа, получение конфигурации при активной подписке).

## 2. Предмет договора
Pineapple VPN предоставляет пользователю сервис защищенного удаленного доступа к российским сервисам из-за границы в объеме выбранного тарифа или доступного trial-периода.

Сервис не предназначен для обхода блокировок и не должен использоваться в противоправных целях.

## 3. Доступ к сервису
Для использования сервиса требуется авторизация через Telegram и использование Telegram MiniApp.

Пользователь получает доступ к функционалу, который доступен в его аккаунте с учетом:

1. статуса подписки или trial;
2. соблюдения правил использования сервиса;
3. технической доступности инфраструктуры.

## 4. Trial-период
1. Trial предоставляется однократно.
2. Базовая длительность trial — 3 дня.
3. При корректном реферальном сценарии trial может быть увеличен (например, до 7 дней).
4. Trial активируется пользователем вручную через интерфейс (не автоматически).
5. Повторная активация trial не предоставляется.

## 5. Тарифы, кошелек и оплата
1. Доступ к платным периодам предоставляется по тарифам, отображаемым в интерфейсе сервиса.
2. Пользователь пополняет внутренний кошелек через доступные платежные методы.
3. Списание за тариф происходит из баланса кошелька.
4. Минимальная сумма пополнения определяется правилами сервиса и валидацией на стороне backend.
5. При недостаточном балансе покупка тарифа недоступна.

## 6. Продление, окончание и прекращение доступа
1. При активной подписке срок доступа действует до даты окончания оплаченного периода.
2. После окончания срока подписки доступ к VPN-функциям может быть ограничен.
3. При нарушении правил использования оператор вправе ограничить доступ полностью или частично.
4. Если доступ ограничен из-за нарушений, оператор вправе отказать в продолжении обслуживания.
