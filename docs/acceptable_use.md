# Правила использования сервиса (Acceptable Use Policy) Pineapple VPN

Версия: 2026-03-15  
Дата вступления в силу: 15.03.2026

## 1. Назначение сервиса
Pineapple VPN предназначен для защищенного удаленного доступа к российским сервисам из-за границы.

Сервис не предназначен для обхода блокировок и не должен использоваться как инструмент противоправной деятельности.

## 2. На кого распространяются Правила
Настоящие Правила обязательны для всех пользователей Pineapple VPN, включая использование через Telegram-бот и Telegram MiniApp.

Использование сервиса означает согласие пользователя с Правилами.

## 3. Разрешенное использование
Разрешается использовать сервис для личных и рабочих задач при соблюдении законодательства и документов Pineapple VPN, в том числе:
- доступ к личным кабинетам банков;
- доступ к государственным сервисам;
- доступ к рабочим и корпоративным системам;
- доступ к иным легальным онлайн-сервисам.

## 4. Запрещенные действия
Пользователю запрещается:

1. использовать сервис в целях, противоречащих законодательству РФ или страны пребывания;
2. осуществлять сетевые атаки, сканирование портов, эксплуатацию уязвимостей, подбор паролей;
3. распространять вредоносное ПО, заниматься фишингом, мошенничеством или социальной инженерией;
4. использовать сервис для массовых спам-рассылок, абузных запросов и иных злоупотреблений;
5. пытаться обойти технические ограничения, лимиты, защитные механизмы сервиса;
6. использовать сервис для нарушения прав третьих лиц, включая авторские и смежные права;
7. передавать доступ третьим лицам в форме, нарушающей условия сервиса;
8. перепродавать доступ к сервису без прямого разрешения оператора;
9. предпринимать действия, которые приводят к деградации стабильности, безопасности и доступности инфраструктуры.

Оператор вправе применять технические и организационные меры против злоупотреблений, включая:

- мониторинг подозрительной активности;
- временную деактивацию профиля;
- ограничение выдачи или обновления конфигураций;
- иные разумные меры безопасности.

Применение таких мер не считается нарушением обязательств оператора, если они направлены на защиту сервиса и пользователей.

1. Пользователь несет ответственность за все действия, совершенные через его аккаунт, включая действия с использованием выданной конфигурации.
2. Пользователь обязан обеспечить конфиденциальность своего доступа к Telegram-аккаунту и устройствам.
3. При подозрении на компрометацию доступа пользователь обязан прекратить использование сервиса и обратиться в поддержку.

## 7. Реакция на нарушения
При выявлении нарушения оператор вправе применить одно или несколько мер:
1. предупреждение;
2. временное ограничение функций;
3. отключение активного доступа;
4. блокировка аккаунта;
5. аннулирование начислений по реферальной программе;
6. отказ в повторном обслуживании;
7. передача информации в уполномоченные органы в случаях, предусмотренных законом.

Меры применяются по усмотрению оператора с учетом тяжести и повторяемости нарушений.

1. При серьезном нарушении правил оператор вправе приостановить или прекратить доступ до окончания оплаченного периода.

3. Вопросы о возможном восстановлении доступа рассматриваются индивидуально через поддержку.

К нарушениям в реферальной программе относятся, в частности:
